8.1 스프링의 정의
=
- **자바 엔터프라이즈 개발을 편하게 해주는 오픈소스 경량급 애플리케이션 프레임워크**




### 애플리케이션 프레임워크
- 특정 계층이나, 기술, 업무 분야에 국한되지 않고 애플리케이션의 전 영역을 포괄하는 범용적인 프레임워크
- J2EE 애플리케이션 설계와 개발 전략을 다룬 책에서 3만 가량의 샘플 애플리케이션이 그 시초
- `항상 프레임워크를 기반으로 접근하라`를 말하기 위해 예시 프레임워크로 제공한 것
- 이 책이 자바 엔터프라이즈 개발의 전 계층에 등장하는 기술과 애플리케이션의 전 영역ㅇ에 대한 효과 적인 설계와 개발 기법을 다뤘기 때문에 예제 프레임워크도 모든 분야 포괄
- 따라서 스프링도 종합적인 애플리케이션 프레임워크가 됨

- 전 영역을 관통하는 일관된 프로그래밍 모델과 핵심 기술을 바탕으로 해서 각 분야의 특성에 맞는 필요를 채워줌

### 경량급
- 불필요하게 무겁지 않다는 의미
- EJB(Enterprise JavaBean)와 대비되도록 쓴 표현
- EJB
  - 개발환경, 운용서버, 개발과 빌드, 테스트 과정 등 매우 무거운 코드를 가짐
  - 무겁고 느리고 고가의 자바 서버(WAS)가 필요했음
  - 툴없이는 다루기 힘든 설정파일 구조, 까다로운 패키징, 불편한 서버 배치
- 스프링
  - 단순 서버환경인 톰캣, 제티에서도 완벽하게 동작
  - 단순한 개발툴과 기본적인 개발 환경에서도 개발 가능
  - 복잡한 WAS 없이도 서블릿 컨테이너만으로 충분
  - 단순한 환경에서도 EJB의 고급 기술을 대부분 사용 가능
- EJB 전체 기능을 다 설치해야 함 vs 스프링은 필요한 것만 쓸 수 있게해서 가벼움
- EJB는 EJB 서버에 의존적 vs 스프링은 순수 자바로 만들어져 서버환경에 의존하지 않음

### 자바 엔터프라이즈 개발을 편하게
- EJB의 목표 : 로우레벨 기술에 많은 신경을 쓰지 않으면서도 비즈니스 로직을 빠르고 효과적으로 구현하고자 함
- but 더 복잡해짐(EJB 규격에 맞추느라 코드가 오히려 복잡해짐)
- 스프링은 복잡성 없이 이러한 목표를 실현

### 오픈소스
- 아파치 라이선스 버전 2.0
  - 상업적 목적으로 사용 가능
  - 스프링을 사용한다는 점, 원 저작자를 밝혀야 함
  - 패키징 시 라이선스 정보 포함
  - 소스코드 수정 가능
- 장점 : 빠르고 유연한 개발
- 단점 : 지속적이고 안정적인 개발의 불확실성
- 자바는 기업의 가장 중요한 핵심 업무를 관장하는 엔터프라이즈 시스템의 개발에 사용되므로 불확실성을 줄이기 위해 스프링소스라는 전문 기업이 관리
